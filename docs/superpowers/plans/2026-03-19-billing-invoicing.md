# Billing, Invoicing & Subscription Management — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement full billing lifecycle — three billing types (Monthly/Annual/One-Time+AMC), invoice management with Indian GST, subscription lifecycle, and payment history across backend + mobile + web.

**Architecture:** Backend-first approach. Phase 1 builds the Prisma schema migration + pricing calculation service. Phase 2 builds invoice CRUD + generation + PDF + email. Phase 3 builds subscription management. Phase 4 builds payment history. Phase 5 wires all frontend screens. Each phase produces independently testable software.

**Tech Stack:** Express/Prisma/PostgreSQL (backend), Expo/React Native (mobile), Vite/React (web), TanStack React Query, pdfkit (PDF generation), Nodemailer (email)

**Spec:** `docs/superpowers/specs/2026-03-19-billing-invoicing-design.md`

**Shorthand Paths:**
- `M/` = `mobile-app/src/`
- `W/` = `web-system-app/src/`
- `B/` = `avy-erp-backend/src/`
- `P/` = `avy-erp-backend/prisma/`

```sql
-- Generate sequential invoice numbers for existing invoices
WITH numbered AS (
  SELECT id, ROW_NUMBER() OVER (ORDER BY "createdAt") as seq
  FROM invoices WHERE "invoiceNumber" IS NULL
)
UPDATE invoices SET
  "invoiceNumber" = CONCAT('INV-', EXTRACT(YEAR FROM "createdAt"), '-', LPAD(numbered.seq::text, 4, '0')),
  "invoiceType" = 'SUBSCRIPTION',
  "subtotal" = COALESCE(amount, 0),
  "totalTax" = 0,
  "totalAmount" = COALESCE(amount, 0),
  "lineItems" = '[]'::jsonb,
  "gstNotApplicable" = true
FROM numbered WHERE invoices.id = numbered.id;

-- Update nextInvoiceSeq in platform config
INSERT INTO platform_billing_config (id, "nextInvoiceSeq", "updatedAt")
VALUES (gen_random_uuid(), (SELECT COUNT(*) + 1 FROM invoices), NOW())
ON CONFLICT DO NOTHING;
```
