📋 Goals & OKRs — How it works

Goal setting is the first step of the appraisal cycle. Each employee sets 4–8 SMART goals (per Config C1) across KRA categories. KRA goals are weighted and scored; achievement is auto-calculated from target vs actual. Goals cascade top-down: Company OKR → Dept OKR → Individual goal.
Key Fields
🎯
Goal / KRA Title
SMART goal statement. Measurable. Should define "done" clearly. Pre-populated from Config C1 template library.
Text
⚖️
Weightage %
Sum of all employee goals must equal 100%. System validates on save. KRA goals contribute 60% of total score by default (Config C1).
Number (auto-validate)
🏭
Auto-Pull from Production/CRM
When enabled in Config C0, OEE, revenue, and other operational metrics auto-populate actual values — eliminating manual entry for shop-floor and sales employees.
System Integration
Smart Interactions
Click any goal row → form pre-fills with all values for editing. RAG status auto-updates based on actual vs target.
Weightage-adjusted KRA contribution shown live: 30% weight × 104% achievement = 31.2/30 pts. Over-achievement capped at 120% per Config C1.
Mid-year edit window: changes outside the window (after Oct 15) require manager approval before saving.

📋 Appraisal Cycle — How it works

The appraisal uses a 7-step workflow configured in Config C2. Final score = KRA 70% + Competency 30% (adjustable). Bell-curve calibration enforces 5/15/60/15/5 distribution. Increment % and promotion rules are automatically applied per rating (R1: 0%, R2: 3%, R3: 7%, R4: 12%, R5: 20%).

📋 360° Feedback — How it works

360° is multi-rater, anonymous feedback configured in Config C3. Rater types: Self, Manager, Skip-Level, Peers (3), Subordinates (3), Cross-Function (2). Minimum 3 responses required before a report is shown. Feeds into competency score when enabled.
Smart Interactions
Full anonymity — all rater identities hidden from rated employee. Manager visible option available in Config C3.
If fewer than 3 responses are received, the 360° report is suppressed to protect anonymity (Config C3 setting).
Open-text questions (Strength, Improvement, Would Work Again) appear as anonymised verbatims in the employee's development report.

📋 Skills & Training — How it works

Skills are rated on a 1–5 proficiency scale (Config C4). The heatmap highlights gaps (≤2) across role-required skills. When gaps are detected, the Auto-Nominate button creates training nominations in one click. Certification expiry tracking is enabled per Config C4 toggles.
Smart Interactions
Skill Level 1 (Gap) — auto-highlighted in red. One-click auto-nomination creates training request for the specific gap skill, pulling the recommended programme from Config C4 library.
On cert upload, skill level auto-updates to 3+ (Practitioner). Expiry date is tracked; HR and employee notified at 30-day mark.
If mandatory cert is expired, promotion workflow is blocked automatically (Config C4 toggle: "Block Promotion if Mandatory Cert Expired").

📋 Succession Planning — How it works

Succession planning maps critical roles (from Config C5 library) to employee bench readiness. The 9-box grid is auto-generated from appraisal performance (x-axis) and HR-assessed potential (y-axis). HiPo programme is triggered for employees in the top 3 cells.
Smart Interactions
9-box positions auto-update after each appraisal cycle close. Performance axis is from the final rating; potential axis is manually updated by HR.
Retention Risk alert fires if a HiPo employee hasn't had a check-in conversation logged in 90 days (Config C5 toggle: "Retention Risk Alerts").
Critical roles marked "Zero Vacancy" (DRI, Statutory) are highlighted in red when the successor bench is empty or has no "Ready Now" candidate.
